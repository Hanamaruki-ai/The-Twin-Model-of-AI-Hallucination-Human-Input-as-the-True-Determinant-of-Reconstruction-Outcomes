■ 2. Hallucination Is Not a Model Problem — It Is a Pre-Process Problem

（ハルシネーションはモデルの欠陥ではない――前工程問題である）

EN

From the TPS perspective, hallucination is never a “defect generated at the output stage.”
Defects originate upstream, from incomplete or inconsistent pre-process inputs.

Thus:

Hallucination = natural divergence when preconditions are not standardized

No algorithmic patch can permanently fix upstream inconsistency

The user, not the model, is responsible for standardizing the upstream process

This repo implements input-design methods that function as AI-equivalent 前工程保証
(Pre-process Assurance), resulting in predictable output zones (“Safe Zones”).

This is precisely how TPS prevents defects before they occur.
We simply transferred that philosophy to AI systems.

JP

TPSの視点では、ハルシネーションは「出力工程で発生した不良」ではありません。
不良は必ず 前工程（入力）での乱れ から生まれます。

つまり：

ハルシネーション＝前提条件が整っていないときの自然なズレ

アルゴリズム改修で前工程の乱れは永久に消せない

標準化するのは“モデルではなくユーザー側の入力”
